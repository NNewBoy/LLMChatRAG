##一
我想撰写一个软件SPEC，在以下需求的基础上进行优化：
1、软件功能：实现一个同时支持普通聊天Agent和RAG功能的在线网站；
2、软件架构：
- 后端负责实现Agent、RAG与数据记录，支持logging日志打印
- 前端负责对话与交互
3、软件框架：前端Vue3+Vite+UI组件element-plus，后端Python+FastAPI+SQLite+Faiss+DeepAgents+LlamaIndex
4、Agent实现：
- 通过DeepAgents框架调用LLM模型，LLM模型使用LangChain的init_chat_model函数连接
- 支持多个技能SKILL.md的使用，由DeepAgents自己决定使用什么skill
- 支持长期记忆（SQLite保存），连续对话，停止生成对话，重新生成对话，挑选历史回答追问；支持联网查询等基础功能；
- 模型支持多模态时，可上传图片，图片转为 LLM 可识别的多模态消息（base64 / URL）
- 实现意图识别：判断是普通聊天还是RAG，RAG作为工具由Agent调用
- 支持环境变量配置的变量：LLM模型、APIKEY、LLM API Base URL、SQLite数据库地址配置路径、服务器HOST与PORT
5、RAG实现：
- 通过LlamaIndex框架实现，实现过程：
-- 文档解析：支持将PDF、Word、HTML、TXT等格式转换为纯文本（保留元数据）
-- 文档分块 (Chunking)：将长文档切分为适合检索的小块(保留来源元数据以便回溯)
-- 向量化 (Embedding)：使用Embedding模型将文本块转换为向量
-- 向量存储：将向量存入向量数据库FAISS
-- Query改写：使用LLM改写或扩展用户问题，提高检索召回率
-- 检索（Retrieval）：使用向量检索（Embedding模型）计算Query向量与文档块向量的相似度，返回Top-K结果；也可以使用关键词检索（BM25），根据两类检索按比例计算出混合相似度返回Top-K结果
-- 重排序（Reranking）：使用LLM模型（与4的LLM模型一致）对Top-K结果精排，选出最相关的Top-N
-- Prompt 构建：将检索到的文档块拼接到Prompt中，作为LLM的上下文
-- 答案生成：LLM模型（与4的LLM模型一致）基于Prompt生成最终答案。
- Bad Case：每次回答，前端可以选择正确与否，生成错题集(SQLite保存)
- 支持环境变量配置的变量：Embedding模型、Embedding APIKEY、Embedding API Base URL、错题集数据库地址配置路径、FAISS数据库地址配置路径、是否启动Query改写、是否启动混合检索（默认仅使用向量检索）、是否启动重排序
6、前端设计：
- 分为两个模式：普通对话模式、RAG对话模式
- 普通对话模式：
-- 用户输入问题，调用后端接口返回结果，对话流式输出采用SSE格式，注意要在前端展示LLM的思考过程；
-- 可查看历史对话场景列表，可创建新对话场景（从零开始对话，不受其他对话场景影响）；
-- 对话可编辑：点击停止对话，删除某一次对话（从对话历史中移除），挑选某一对话重新生成对话（比该对话新的对话历史移除出当前对话），挑选某一对话选择追问
-- 若后端的模型支持多模态，则可上传图片并对话
-- 可询问关于RAG文档中的数据，由Agent做意图识别，识别为RAG中数据则直接调用RAG返回，否则继续普通对话
-- 支持选择LLM模型（由后端提供可选模型列表-告知前端是否支持多模态），是否启动意图识别（启动则加载RAG工具，不启动则不进行意图识别并且无需加载RAG工具）
- RAG对话模式：
-- 分为三个功能：对话功能，上传文档生成RAG数据，错题集编辑
-- 对话功能：
--- 用户输入问题，后端无需意图识别直接调用RAG返回结果，对话流式输出采用SSE格式，注意要在前端展示工具调用及思考过程；
--- 可查看历史对话场景列表，可创建新对话场景（从零开始对话，不受其他对话场景影响）；
--- 对话可编辑：点击停止对话，点击重新生成对话，挑选某一对话选择追问，可以选择回答正确与否
-- 上传文档生成RAG数据：
--- 用户可以选择PDF、Word、HTML、TXT等格式的文档，上传后由后端生成RAG数据
--- 展示所有上传文档的状态（轮询：已完成、解析中等），用户可以删除文档
-- 错题集编辑
--- 根据对话中选择的错误答案生成错题集
--- 用户给错题编写正确的答案，确认是否给LLM传递范例
-- 前端也支持选择Embedding模型（由后端提供可选模型列表）、是否启动Query改写、是否启动混合检索（默认仅使用向量检索）、是否启动重排序
-- UI设计兼容PC端与移动端
7、文档：
- DeepAgents框架文档：https://docs.langchain.com/oss/python/deepagents/overview
- 硅基流动嵌入向量请求文档：https://api-docs.siliconflow.cn/docs/api/embeddings-post
